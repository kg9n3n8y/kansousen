# 札跡 改修仕様書 v1.0

## 1. 概要

| 項目 | 内容 |
|------|------|
| アプリ名 | 札跡（ふだあと） |
| 目的 | かるた試合後の取札・取られた札を陣形上に記録・分析する |
| 配信形態 | PWA（GitHub Pages: `/kansousen/`） |
| 技術スタック | Vite + React + TypeScript + Tailwind CSS + vite-plugin-pwa |

## 2. 確定要件

### 2.1 一括入力モード（既存UIの作り直し）

- ボタンは「一括入力」1つのまま
- `KIMARIJI_LIST` 順（む→す→め→ふ→さ→ほ→せ→…）で100札を巡回
- **ステップ式UI**（場所 → 何字 → 誰が）で各選択後に自動進行
- 空札（スキップ）で登録削除して次へ
- 既存登録がある札は値をプリフィル、変更なしでも次へ進める
- 前の札へ戻って修正可能

### 2.2 編集・削除

- 盤面の札タップで「編集」「削除」アクションメニューを表示
- 編集: 既存ウィザード（取札を追加ダイアログ）を編集モードで開く
- 削除: アプリ内確認モーダル（`window.confirm` は使わない）

### 2.3 過去データ保存（ローカル）

- 保存単位: **1試合 = 1盤面**
- 同時編集: **1試合のみ**（作業中盤面、他は一覧から読み込み）
- 保存件数: **上限なし**
- タイトル: 現行の「画像に表示するメモ」フィールドを試合タイトルとして使用
- ストレージ: `localStorage`（キー `kansousen-history-v1`）
- 旧データ（`kansousen-state-v1`）の移行: **不要**

```typescript
interface MatchRecord {
  id: string;
  title: string;
  createdAt: string;  // ISO 8601
  updatedAt: string;
  entries: Entry[];
}

interface HistoryState {
  currentMatchId: string;
  matches: MatchRecord[];
}
```

#### UI

- 試合一覧: 日付・タイトル・登録札数
- 新規試合: 空盤面で開始
- 試合を開く: 一覧から選択して盤面に復元
- 自動保存: 編集のたびに現在の試合を更新
- 試合削除: 確認モーダル後に削除

### 2.4 UIデザイン

- テイスト: **かるたの札・畳・墨色**を基調とした落ち着いた伝統的和風
- 質感: ネイティブアプリ風（カードUI、ボトムシート、適切な余白）
- レスポンシブ: スマホ / タブレット / PC 対応
- CSS: Tailwind CSS

### 2.5 PWAインストール案内

- 表示: **初回訪問時のみ**（`localStorage` フラグ）
- PWA（`display-mode: standalone`）起動時は**非表示**
- 「もう一度見る」リンク: **不要**
- OS/ブラウザ別の手順を表示（iOS Safari / Android Chrome / Desktop）

### 2.6 PWA自動更新

- `vite-plugin-pwa` で Service Worker 管理
- HTML: network-first、ハッシュ付きアセット: cache-first
- 新バージョン検知時に更新トーストを表示、ユーザー操作でリロード
- ページフォーカス時に `registration.update()` を実行

## 4. 実装フェーズ

| Phase | 内容 |
|-------|------|
| 1 | Vite + React + TS + Tailwind + PWA 基盤 |
| 2 | コア機能移行（盤面・ウィザード・画像出力） |
| 3 | 一括入力UI作り直し |
| 4 | 編集/削除メニュー + 試合履歴 |
| 5 | 和モダンUI仕上げ + PWA案内 + 自動更新 |

## 5. 非機能要件

- アクセシビリティ: キーボード操作、ARIA属性
- GitHub Pages デプロイ: `base: '/kansousen/'`
